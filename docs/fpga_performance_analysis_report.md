# DFS 分布式文件系统性能分析报告

## FPGA AES 加速集成与性能预测

---

## 1. 项目背景

### 1.1 研究目标

将 FPGA AES 加速器集成到分布式文件系统 (DFS) 中，对比分析三种实现方案的性能：

| 方案 | 描述 | 特点 |
|------|------|------|
| **OpenSSL** | 纯软件实现 | CPU + OpenSSL AES-NI 硬件加速 |
| **FPGA** | 纯硬件实现 | Xilinx U200 FPGA 加速卡 |
| **SmartNIC** | 智能网卡实现 | FPGA 集成网卡，零拷贝 |

### 1.2 系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                        客户端 (DFC)                          │
│  ┌─────────────────────────────────────────────────────┐    │
│  │              ThreadPool (4 工作线程)                  │    │
│  │   ┌───────┐ ┌───────┐ ┌───────┐ ┌───────┐          │    │
│  │   │Thread1│ │Thread2│ │Thread3│ │Thread4│          │    │
│  │   └───┬───┘ └───┬───┘ └───┬───┘ └───┬───┘          │    │
│  └───────┼─────────┼─────────┼─────────┼───────────────┘    │
│          ▼         ▼         ▼         ▼                    │
│      Server1   Server2   Server3   Server4                  │
└─────────────────────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────────────────────┐
│                    服务器端 (DFS)                            │
│  ┌─────────────────────────────────────────────────────┐    │
│  │         主进程 (监听端口) + fork() 多进程             │    │
│  └─────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────┘
```

## 5. 瓶颈分析

### 5.1 延迟分解

```
总延迟 = 网络延迟 (85%) + 加密延迟 (10%) + 其他开销 (5%)
```

### 5.2 瓶颈识别

| 因素 | 影响 | 说明 |
|------|------|------|
| **网络 I/O** | 主要瓶颈 | 4 节点分布式存储，延迟 200-500ms |
| 加密计算 | 次要 | OpenSSL AES-NI 已很快 |
| FPGA PCIe 传输 | 小文件瓶颈 | 传输开销占比高 |

### 5.3 FPGA 优势受限原因

1. **网络延迟占主导**: 加密时间远小于网络传输时间
2. **PCIe 传输开销**: 小文件场景下开销占比高
3. **仅支持加密**: 解密仍使用 CPU，无法完全卸载

---

## 6. SmartNIC 预测分析

### 6.1 SmartNIC 优势

| 特性 | 说明 | 性能提升 |
|------|------|---------|
| **零拷贝** | 数据直接在网卡内存处理 | 减少 PCIe 传输 |
| **CPU 卸载** | 主机 CPU 不参与加密 | CPU 使用率 <1% |
| **低延迟** | 避免主机协议栈开销 | 减少 10-20% 延迟 |
| **线速处理** | 支持 100+ Gbps | 充分利用网络带宽 |

### 6.2 预测模型

```python
# 大文件 (≥1MB): 网络延迟占 85%
smartnic_latency = cpu_latency × (network_ratio × 0.95 + other_ratio)

# 中等文件 (64KB-1MB): 网络延迟占 70%
smartnic_latency = cpu_latency × (network_ratio × 0.90 + other_ratio)

# 小文件 (<64KB): 固定开销占主导
smartnic_latency = cpu_latency × (network_ratio × 0.85 + other_ratio × 0.8)
```

### 6.3 预测结果

| 指标 | OpenSSL | FPGA | SmartNIC |
|------|---------|------|----------|
| PUT 吞吐量提升 | 基线 | +5-6% | **+15-25%** |
| GET 吞吐量提升 | 基线 | -1-2% | **+10-20%** |
| CPU 使用率 | 3-4% | 2-3% | **<1%** |
| 内存使用率 | 1.5-2% | 1.4-1.9% | **<0.3%** |
| 延迟降低 | 基线 | 0-5% | **10-20%** |

## 7. 结论与建议

### 7.1 主要发现

1. **当前 FPGA 实现提升有限**: 由于网络 I/O 是主要瓶颈，FPGA 加密优势无法充分发挥

2. **SmartNIC 是更优方案**: 零拷贝和 CPU 卸载能显著提升性能

3. **小文件场景 SmartNIC 优势明显**: 可减少协议栈开销，加速比达 1.7x

### 7.2 优化建议

| 优先级 | 建议 | 预期效果 |
|--------|------|---------|
| 高 | 采用 SmartNIC 方案 | 吞吐量提升 15-25% |
| 中 | FPGA 添加解密支持 | GET 性能提升 |
| 低 | 批量加密优化 | 减少 PCIe 开销 |

### 7.3 适用场景

| 方案 | 适用场景 |
|------|---------|
| OpenSSL | 通用场景，无特殊硬件 |
| FPGA | 加密密集型应用，大文件处理 |
| SmartNIC | 高性能网络，数据中心 |
